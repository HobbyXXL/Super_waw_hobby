🧠 Как работает проект

Пользователь нажимает кнопку на сайте или отправляет запрос через API

Запрос попадает в routes (маршруты)

Routes вызывают services (логику приложения)

Services взаимодействуют с models (таблицами базы данных)

Данные сохраняются в SQLite или кешируются в Redis

Ответ возвращается на фронтенд

app/

🧩 Как всё связано
Запрос → routes → services → models → база данных → ответ

routes — “кнопки” API, принимают запросы

services — бизнес-логика, обработка данных

models — таблицы базы данных

schemas — валидация и сериализация данных

core — основные настройки и утилиты (DB, безопасность, Redis)

🔹 Основные функции
1️⃣ Пользователи

Регистрация, логин, получение информации о себе или другом пользователе

Пароли не хранятся в открытом виде

JWT-токен используется для аутентификации

Эндпоинты:

POST /auth/register
POST /auth/login
GET /users/{user_id}
2️⃣ Хобби

Список всех хобби

Получение хобби пользователя

Используется для рекомендаций и поиска друзей

Эндпоинты:

GET /           # список всех хобби
GET /{user_id}  # хобби конкретного пользователя
3️⃣ Посты

Создание поста (текст, картинка, рейтинг, ссылка на внешнюю статью)

Просмотр ленты

Лайки и комментарии

Кэширование ленты через Redis для ускорения

Эндпоинты:

POST /posts/
GET /posts/
POST /posts/{post_id}/like
POST /posts/{post_id}/comments
4️⃣ Сообщества

Создание и редактирование сообществ

Вступление и выход из сообществ

Просмотр участников

Уникальные slug для ссылок

Эндпоинты:

POST /communities/
POST /communities/{community_id}/join
GET /communities/{community_id}/members
5️⃣ Личные и групповые чаты

Список диалогов: личные и групповые

Пересылка постов из ленты

Создание совместного холста (“доска”)

Поле ввода сообщений и прикрепления файлов

Пока можно использовать заглушки, но backend готов к расширению

6️⃣ Кэширование и производительность

Redis используется для:

кэширования ленты постов

хранения сессий пользователей

Это ускоряет ответы API и уменьшает нагрузку на SQLite

GET — получить данные

POST — создать объект

PUT — изменить объект

DELETE — удалить объект

Уникальные значения: slug для сообществ, email и login для пользователей

Если удалить test.db, база очистится и все начнётся заново
