# 💙 HobbyFriend — backend проекта

Привет! 👋
Это backend (серверная часть) нашего проекта **HobbyFriend** — приложения, где люди могут:

* находить друзей по интересам
* писать посты
* вступать в сообщества
* делиться своими хобби

Я постаралась сделать код максимально понятным и простым 💙

---

# 🧠 Как вообще работает проект

Очень просто:

👉 есть **сервер (FastAPI)**
👉 есть **база данных (SQLite)**
👉 есть **запросы (GET, POST и т.д.)**

📌 Когда пользователь нажимает кнопку на сайте →
📌 отправляется запрос →
📌 сервер что-то делает →
📌 возвращает ответ

# 🧩 КАК ВСЁ СВЯЗАНО

Вот прям цепочка:

```id="2h3f9d"
Запрос → routes → services → models → база данных
```

---

## 🔹 routes (API)

👉 это “кнопки” нашего backend

Пример:

```python id="9m2k1a"
@router.post("/auth/register")
```

👉 значит можно отправить запрос и зарегистрироваться

---

## 🔹 services (логика)

👉 здесь происходит “магия”

Например:

```python id="l8v1a2"
def register(self, data):
```

👉 создаётся пользователь
👉 хэшируется пароль
👉 сохраняется в базу

---

## 🔹 models (таблицы)

👉 это структура базы данных

Пример:

```python id="x0s9q1"
class Post(Base):
    id = Column(Integer, primary_key=True)
    content = Column(String)
```

👉 это значит:

* есть пост
* у него есть текст

---

## 🔹 core

👉 важные настройки

* database.py → подключение к базе
* security.py → пароли и токены

# 🎯 ЧТО ТАКОЕ “ХОББИ” В ПРОЕКТЕ

Хобби — это интересы пользователя

Примеры:

* рисование 🎨
* программирование 💻
* спорт ⚽

---

## 👉 Что можно делать:

### ✔ получить список хобби

```http
GET /
```

---

### ✔ посмотреть хобби пользователя

```http
GET /{user_id}
```

---

📌 Это нужно чтобы:

👉 находить людей с одинаковыми интересами
👉 строить рекомендации
